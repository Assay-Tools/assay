# Outreach Templates

Templates for reaching out to package maintainers. **AJ reviews and approves all outbound communications before sending.**

---

## Template 1: High Scorer Congratulations (AF 80+)

**Subject**: Your [package] scored [score]/100 on agent-friendliness

**Use when**: Package has AF score 80+. Goal is relationship-building and awareness, not a hard sell.

---

Hi [name],

I run Assay (https://assay.tools), an independent directory that scores MCP servers, APIs, and SDKs on how well they work with AI agents.

Your package [package-name] scored [score]/100 on agent-friendliness — that puts it in the top tier of the [category] category. The score reflects strong [mention 1-2 specific strengths, e.g. "documentation quality and error handling"].

Full evaluation: https://assay.tools/packages/[package-id]

We're building this directory to help agents and developers find tools they can trust. If you'd like a detailed breakdown of your scores across all 14 sub-components, or if anything in the evaluation looks off, let me know — we want to get this right.

Best,
AJ

## Usage Notes

- **Personalize every message** — reference specific scores, strengths, and weaknesses. Generic messages get ignored.
- **Don't batch-send** — each outreach should feel like a 1:1 message, because it is.
- **Prioritize by**: packages with 55-75 scores (room to improve = report value), high GitHub stars, active development.
- **Track responses** — note who replies, what they care about, and what objections come up.
- **Respect "no"** — if someone asks not to be contacted, respect it immediately.
